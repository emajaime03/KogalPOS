Arquitectura del Sistema KogalPOS

Resumen de Alto Nivel

El sistema sigue una arquitectura en capas estricta, utilizando el patrón Facade (RequestService) para orquestar la lógica de negocio y mantener el desacoplamiento entre la UI y la BLL.

Componentes y Patrones Key

UI Presentación

Implementada con DevExpress para WinForms.

  • Template Method: Uso de frmBaseABM que define el esqueleto de un ABM.
  • Observer Pattern: Los formularios implementan IObserver para reaccionar a cambios.

Services Facade & Core

Punto de entrada único para la lógica de negocio.

  • Facade Pattern: RequestService oculta la complejidad.
  • Singleton: Servicios transversales y BLLs son Singletons.
  • Request-Response: Comunicación tipada mediante DTOs.

BLL Logica de Negocio

Contiene reglas, validaciones y control de transacciones.

DAL Acceso a Datos

Abstracción de la base de datos SQL Server mediante Unit of Work y Repository.
