Arquitectura del Sistema KogalPOS

Resumen de Alto Nivel

El sistema sigue una arquitectura en capas estricta. La UI llama directamente a los BLLs (Singletons) mediante objetos Request/Response tipados, manteniendo type-safety sin capas intermedias innecesarias.

Componentes y Patrones Key

UI Presentación

Implementada con DevExpress para WinForms.

  • Template Method: Uso de frmBaseABM que define el esqueleto de un ABM.
  • Observer Pattern: Los formularios implementan IObserver para reaccionar a cambios.

Services Servicios Transversales

Servicios de soporte compartidos por todas las capas.

  • Singleton: BLLs y servicios transversales son Singletons.
  • Request-Response: Comunicación type-safe mediante DTOs tipados.
  • LoggerService: Registro de eventos y excepciones.
  • CryptographyService: Hashing de contraseñas.

BLL Logica de Negocio

Contiene reglas, validaciones y control de transacciones.

DAL Acceso a Datos

Abstracción de la base de datos SQL Server mediante Unit of Work y Repository.
